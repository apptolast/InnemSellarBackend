§Por que #[tokio::main]

Rust no tiene async runtime integrado (a diferencia de Dart que tiene su event loop). #[tokio::main] inicializa el runtime async de tokio, que gestiona las tareas concurrentes (conexiones HTTP, queries a DB, etc.). Es como arrancar el event loop de Dart, pero de forma explicita.

§Por que async fn main()

async permite usar .await dentro de la funcion. Cada .await cede el control al runtime para que pueda ejecutar otras tareas mientras esperamos I/O (red, disco). Igual que en Dart con async/await, pero en Rust el compilador verifica en tiempo de compilacion que no haya data races.
