Inicializa la conexion a PostgreSQL via SeaORM.

§Cambio vs version anterior (SQLx directo)

Antes: PgPoolOptions::new().connect(&url) → devuelve PgPool Ahora: Database::connect(&url) → devuelve DatabaseConnection

SeaORM gestiona el pool internamente (usa SQLx por debajo). DatabaseConnection es Clone y Send + Sync — seguro para compartir entre hilos y handlers.

§Por que seguimos usando expect() aqui

Es codigo de inicializacion. Si no hay BD, el servidor no puede funcionar. Mejor fallar rapido con mensaje claro que arrancar en estado inconsistente.
